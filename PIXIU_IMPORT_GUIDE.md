## 🔄 交易类型处理

### 普通交易
- **流入金额 > 0** → INCOME（收入）
- **流出金额 > 0** → EXPENSE（支出）

### 转账交易
- **格式**：`账户A → 账户B`
- **处理**：创建两条交易记录
  - 从 A 账户出账（EXPENSE）
  - 到 B 账户入账（INCOME）
- **分类**：自动归类为"转账"
- **示例**：
  - `农行2917 → 支付宝余额` → 农行出账 + 支付宝入账
  - `支付宝余额 → 微信零钱` → 支付宝出账 + 微信入账

### 空分类
- 自动归类为"其他"

## 🐛 常见问题

### Q: 导入后数据重复了怎么办？
A: 系统不会自动去重。如果需要清理重复数据，请：
1. 查看交易列表
2. 手动删除重复项
3. 或联系管理员

### Q: 转账为什么被跳过了怎么办？
A: 转账现在会被导入！系统会自动识别 `账户A → 账户B` 格式，创建两条交易记录（出账 + 入账）。

### Q: 如何修改账户/分类映射？
A: 在导入第3步可以修改映射关系。导入后也可以在交易详情中修改。

### Q: 导入失败了怎么办？
A: 
1. 检查 CSV 文件格式
2. 查看错误日志：`tail -f log/development.log`
3. 尝试重新上传

### Q: 可以导入多个文件吗？
A: 可以。每次导入都会创建新的交易记录。建议分阶段导入（按月份）。

## 🔧 技术细节

### 导入流程
1. **上传** → 验证文件格式和大小
2. **预览** → 统计有效/无效/转账交易
3. **映射** → 配置账户和分类对应关系
4. **导入** → 逐行处理，创建交易记录
5. **完成** → 显示导入统计

### 错误处理
- 单条交易失败不影响其他交易
- 所有错误记录在日志中
- 导入完成后显示错误统计

### 性能优化
- 使用 `find_each` 避免内存溢出
- 批量创建账户和分类
- 事务隔离，单条失败不回滚整体
