# ADR-002: 负数支出退款约定

**日期**: 2026-04-14
**状态**: 已完成
**PR**: #107

## 背景

退款交易有两种来源：
1. **买家退款**：已支出的钱退回（如退货）
2. **卖家退款**：已收入的钱退回（如售出后退款给买家）

旧系统将所有退款统一存为 `income` 类型，导致分类统计时退款被计入收入，而非抵消原支出。

## 决策

采用"金额符号 + kind"组合约定：

| 场景 | kind | 金额符号 | 说明 |
|------|------|---------|------|
| 正常支出 | expense | 正数 | 花钱 |
| 买家退款 | expense | 正数 | 支出减少（正数表示退回） |
| 正常收入 | income | 正数 | 赚钱 |
| 卖家退款 | income | 负数 | 收入减少（负数表示退回） |

### 核心规则
- `kind` 始终与分类类型一致
- 金额符号区分正常交易和退款
- 统计查询按 `kind` 过滤，退款自然抵消

## 影响

- 分类统计中退款正确抵消原类别
- 不再需要单独的退款类型
- 需要修正历史数据中 kind 不匹配的记录（已完成，923 条）
